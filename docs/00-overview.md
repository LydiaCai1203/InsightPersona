# AI 编程工具用户画像分析方案

## 项目概述

基于约 40 万条用户对话记录，构建一套从特征提取到用户画像的完整分析管线（pipeline），最终产出可直接指导产品迭代、增长运营和商业化决策的用户洞察。

## 核心目标

| 目标 | 对应输出 |
|------|---------|
| 我们有哪几种用户 | 5-10 个自然用户群落画像（Persona） |
| 什么让用户留下来 | 激活公式 + 关键时刻地图 |
| 谁会付费、付多少 | Free/Pro/Enterprise 分层模型 |
| 产品哪里好、哪里差 | 需求 × 成功率 的产品能力矩阵 |
| 用户怎么变化的 | 3-5 条典型用户旅程 |
| 用户在做什么生意 | 垂直行业机会表（如量化交易 → K 线数据供给） |
| 谁能拍板买单 | B2B 采购信号矩阵 |

## 数据规模假设

- 总对话 turn 数：约 40 万条
- 预估 task 数：约 8-15 万（假设每个 task 平均 3-5 个 turn）
- 用户数：待确认

## 方法论

**不使用** embedding + KMeans 直接聚类（已验证效果差，聚出来的是"任务类型"而非"用户类型"）。

采用 **结构化特征提取 → 分层聚合 → 目的驱动分析** 的路线：

```
原始对话数据
    │
    ├─→ Turn 层：规则/Regex 提取（零成本）
    │
    ├─→ Task 层：LLM 标注（约 $15-22）
    │
    ├─→ User 层：统计聚合（零成本）
    │
    ├─→ 时间维度：演化特征（零成本，基于已有标签做时序聚合）
    │
    └─→ 11 条分析路线 → 业务决策
```

## 设计原则

1. **目的驱动**：每个特征必须映射到至少一个业务决策，不提取"有意思但没用"的维度
2. **分层提取**：Turn 层用规则，Task 层用 LLM，User 层用统计聚合，成本最优
3. **两批标注**：Task 层 19 个字段分两批，Batch 1 做客观事实字段，Batch 2 做推断性字段
4. **容忍不确定**：`unclear` 是合法标签，不强行标注低置信度字段
